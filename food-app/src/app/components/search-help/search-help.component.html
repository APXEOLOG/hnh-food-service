This search support lucene-like queries

Available terms:
  • name - find with food name containing value (case-insensitive). For example: name:roast
  • from - find with ingredient containing value (case-insensitive). For example: from:laurel
  • Fep-based terms (str, cha2, prc1, ...) - find with specific fep value. You can use operators >, <, >=, <=. For example: str2:>0
  • Search negation - you can filter-out some items using term starting with -. For example: -name:spit
  • Available fep codes are: {{ fep }}
Some examples:
name:roast str:>0 int:>0 -wil:>0 - Find any food with "roast" in the name, with any STR feps > 0, any INT feps > 0 and no WIL feps
from:milk agi2:>0 - Find any food made from milk with AGI+2 feps
name:roast -name:spit prc2:>0 - Find any roasted, but not spitroasted food with PRC+2 feps
You can also use OR operators like in lucene. By default all terms are combined via AND operator
